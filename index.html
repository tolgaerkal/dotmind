Mind Primitives

Five questions every mind must answer

Together they reconstruct full context for any LLM session — instantly, without re-explanation. Remove one and there's a meaningful gap. Keep all five and any agent or human is grounded from the first message.

The primitives are agnostic. They work for a software project, a government programme, a department, or a person. The same five questions, the same repo-based structure, the same instant grounding.

identity
What is this
Purpose, phases, domain, current status. The LLM knows what it is working on.
resources
What governs it
Every document, requirement, standard, and source in play — local, external, or integrated.
authority
What wins conflicts
Explicit trust hierarchy. When sources conflict, the LLM knows exactly who to defer to.
boundaries
What can be touched
What agents and people can read, modify, or act on. Nothing is left ambiguous.
history
What was decided
Append-only decisions and reasoning. The LLM never relitigates or contradicts settled ground.

Who it serves

One mind, three stakeholders

The same five primitives serve everyone on the project — from the same repo, at the same time.

01
Project manager
Maintains the mind conversationally. Every LLM session is instantly grounded. Never re-explains context between sessions or to new collaborators.
02
Dev team & agents
Reads the mind before starting work. Knows what is approved, what can be touched, what is already decided. Tactical AI work stays grounded without interrupting flow.
03
Client
Always-current picture of the project in plain language. Owns the repo. Retains full memory when vendors change. Can query at any point without chasing the team.
Surfaces

Three interfaces, one mind

The repo is the source of truth. CLI and MCP server are the surfaces on top — one for humans, one for agents. Both read from and write to the same underlying mind.

CLI
Human interface
Conversational setup, query, and maintenance. mind init, mind ask, mind update. Built in Node.js, works anywhere.
MCP server
Agent interface
The mind is available as tools natively inside any LLM session. Agents read and write without leaving the workflow. No re-explaining. No cold starts.
Repo
Persistent layer
Versionable, auditable, client-ownable. Neutral ground that survives tool changes, vendor changes, and team changes. Git does all the hard work.
